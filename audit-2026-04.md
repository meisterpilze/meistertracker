## Phase 1 ‚Äî Security

11 findings: 2 HIGH, 6 MED, 3 LOW. No live RCE, no plaintext passwords on disk, no session-cookie disasters. Full draft in `phase-1-security.md`.

### S-01 ‚Äî MCP transport is role-blind; any token = full admin [HIGH] `OPEN`
`mcp-server.js` (whole file ‚Äî no role gates) + `server.js:124-141` (`checkMcpAuth` returns boolean, drops the OAuth token's `userId`) + `server.js:3810-3815` (OAuth `/authorize` GET allows any logged-in user, not just admins).

The MCP tool surface exposes destructive operations (`delete_batch`, `delete_culture`, `delete_mushroom_strain`, `manage_zones { delete }`, `manage_racks`, `update_inventory`, `assign_barcode`, etc.) and never inspects who the caller is. Two paths reach the transport:
1. **Legacy static token** (`db.getMcpToken`) ‚Äî single global API token, anyone holding it gets full admin.
2. **OAuth access token** ‚Äî `/oauth/authorize` requires only a session cookie, so a worker can grant a Claude client an access token, which then enjoys the same admin-level MCP surface.

**Impact**: A worker who installs Claude Desktop and clicks "Allow" escalates to full DB-mutation rights. The web UI gates the same operations behind `requireAdmin` (e.g. `server.js:4721, 5340, 5448, 5538, 5611`).

**Fix sketch**: (a) `checkMcpAuth` returns `{userId, role}` instead of `true/false`; (b) wrap each MCP tool with a required-role check (or a wrapper around `server.tool`); (c) restrict `/oauth/authorize` to admins, or scope OAuth-derived tokens to a read-only tool subset.

### S-02 ‚Äî `/api/webhook/github` crashes server on bad-length signature [HIGH] `OPEN`
`server.js:5921` inside an unguarded `req.on('end', () => { ... })`.

`crypto.timingSafeEqual(Buffer.from(sig), Buffer.from(expected))` throws `RangeError` if the attacker-controlled `X-Hub-Signature-256` header isn't 71 chars. The throw escapes the callback ‚Üí `uncaughtException` handler at `server.js:7083` ‚Üí `process.exit(1)`.

**Impact**: One unauthenticated POST per ~PM2-restart-window keeps the server in a permanent crash loop. Only active when `GITHUB_WEBHOOK_SECRET` is configured (the auto-deploy flow added in #340).

**Fix sketch**:
```js
const sigBuf = Buffer.from(sig);
const expBuf = Buffer.from(expected);
if (sigBuf.length !== expBuf.length || !crypto.timingSafeEqual(sigBuf, expBuf)) {
  jsonErr(res, 401, 'bad signature'); return;
}
```
Also wrap the `req.on('end')` body in `try/catch`.

### S-03 ‚Äî Worker-writeable endpoints that should be admin [MED] `OPEN`
- `server.js:5111` POST `/api/cultures`
- `server.js:5140` POST `/api/cultures/:id/rename`
- `server.js:5167` PATCH `/api/cultures/:id`
- `server.js:5184` DELETE `/api/cultures/:id` ‚Äî orphans `batches.source_id` (no FK cascade in `db.js:2457`)
- `server.js:5386` POST `/api/assets` ‚Äî workers can upsert assets up to `purchasePrice: 100,000,000`
- `server.js:4799` DELETE `/api/scan-log/:id` ‚Äî comment says "by design" but lets workers delete *anyone's* scan entries

**Impact**: A disgruntled worker can wipe lab cultures or fabricate asset purchases. No actor in audit log.

**Fix sketch**: `if (requireAdmin(req, res)) return;` on each. For scan-log delete, mirror the contamination-report ACL at `server.js:5044-5048` (owner-or-admin).

### S-04 ‚Äî Backup encryption: weak password floor + scrypt cost [MED] `OPEN`
`server.js:6326, 6342, 6406` ‚Äî `scrypt` with `N: 32768, r: 8, p: 1` and 8-char password minimum; redundant outer HMAC-SHA256 over an already-AEAD ciphertext using the same key (`server.js:6348`).

**Impact**: An attacker who exfiltrates an `.enc` backup and an 8-char password dictionary can decrypt the entire DB (sessions, OAuth tokens, DuckDNS material).

**Fix sketch**: `N: 131072, r: 8, p: 4` (~1s on a modern server); enforce ‚â• 12-char password with complexity check; drop the redundant HMAC.

### S-05 ‚Äî Session cookies missing `__Host-` prefix; no rotation on login [MED] `OPEN`
`server.js:458-468, 4127`. Cookies are `HttpOnly; SameSite=Strict; Path=/; Secure (https)` ‚Äî solid baseline. Two refinements:
1. No `__Host-` prefix ‚Üí a sibling subdomain on the same eTLD+1 could plant a cookie.
2. `/api/auth/login` issues a new token but doesn't `db.deleteSession` any pre-existing token presented in the request.

**Fix sketch**: Switch cookie name to `__Host-session`; on login, delete the old token before minting the new one.

### S-06 ‚Äî `setupRequired` race lets attacker claim admin [MED] `OPEN`
`server.js:4065-4096`. First-run gate is only `db.countUsers === 0`. On a fresh public deployment, an unauthenticated remote attacker who hits `/api/auth/setup` first becomes admin.

**Fix sketch**: Print a setup token to stdout on first start (require it as a header), or bind setup to `127.0.0.1`/`::1` (mirror the pattern at `server.js:4400-4411`).

### S-07 ‚Äî Print-bridge token compared with `-eq` (not constant time) [MED] `OPEN`
`scripts/print-bridge.ps1:419-421`. Also: bridge accepts unauthenticated `POST /print` if `$Token` is empty (warning is console-only at line 440).

**Fix sketch**: Use `[System.Security.Cryptography.CryptographicOperations]::FixedTimeEquals` (PS 7+) or manual XOR-and-fold; refuse to start if `$Token` is empty AND the bind isn't `127.0.0.1`.

### S-08 ‚Äî Static MCP token: no rotation history, no last-used, ambiguous encoding [MED] `OPEN`
`db.js:3173-3186` + `server.js:132`. `generateMcpToken` overwrites the prior hash on every call; no `last_used`, `created`, `revoked` columns; `Buffer.from(hexString)` is invoked without specifying `'hex'` encoding, so it works only because both inputs are equal-length UTF-8 strings.

**Fix sketch**: Treat the static token as deprecated in favour of OAuth; add audit columns; pass `'hex'` to `Buffer.from` for clarity.

### S-09 ‚Äî Path-traversal sanitiser is platform-dependent [LOW] `OPEN`
`server.js:1479-1483`. `sanitizePart` relies on `path.basename`, which on POSIX treats `\` as a literal. No exploit today; brittle invariant.

**Fix sketch**: Reject any segment containing `/`, `\`, `\0`, or starting with `.` explicitly.

### S-10 ‚Äî `gen-cert.sh` interpolates `$DOMAIN` into OpenSSL config unescaped [LOW] `OPEN`
`gen-cert.sh:34-66`. Local-admin-only attack surface today; would matter if `CERT_DOMAIN` is sourced from untrusted env later.

**Fix sketch**: Whitelist `^[A-Za-z0-9.-]+$` before interpolation.

### S-11 ‚Äî `/api/health` over-shares to authenticated workers + Host-header reflection [LOW] `OPEN`
`server.js:4354-4363, 284`. Verbose health payload (Node version, platform, memory, SSE clients, backup details) goes to *any* authenticated user. `req.headers.host` is reflected into OAuth issuer metadata.

**Fix sketch**: Restrict verbose health to admins; validate `Host` against a configured `PUBLIC_HOSTNAME` env var.
